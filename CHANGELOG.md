# Changelog

[繁體中文](CHANGELOG.zh-TW.md)

This file records user-visible NexDrop changes and follows Keep a Changelog and Semantic Versioning.

## [Unreleased]

## [2.0.4] - 2026-07-22

### Changed

- Improves bilingual documentation, API and realtime boundaries, extension productivity, Flutter send and connection reliability, deployment diagnostics, and Go 1.26 security automation.

## [2.0.3] - 2026-07-22

### Changed

- Updated all CodeQL workflows to 4.37.3 and grouped related Dependabot updates to keep component versions synchronized.
- Updated `docker/setup-buildx-action` to 4.2.0 and `actions/checkout` to 7.0.1.

### Fixed

- Fixed the auto-merge workflow's author matching for `dependabot[bot]` and `github-actions[bot]`, allowing trusted automated updates to merge after required checks pass.
- Resolved conflicts caused by multiple GitHub Actions updates modifying release workflows at the same time.

## [2.0.2] - 2026-07-22

### Changed

- Removed the Web administration console, browser administration APIs, pairing codes, and manual approval flow. Node operations now use the deployment command exclusively.
- Added `nexdrop://join` onboarding to Windows and Android so Node URLs and keys are imported together; device creation now sends the Node key correctly.
- Release workflows now publish the `major.minor` container tag derived from the product version instead of always publishing `1.0`.
- Trusted generated dependency PRs merge automatically after required checks pass; functional PRs must include matching version and release metadata.

### Fixed

- Fixed Windows tray, connection, and background services continuing to run after closing the window or choosing full exit.
- Fixed device onboarding, page layout, error dismissal, lifecycle cleanup, and load tests that still depended on the retired approval API.

## [2.0.1] - 2026-07-19

### Fixed

- Fixed upgrades from 1.x to 2.0 leaving existing `.env` files without Node and Web OTP keys, which produced blank values after updating.
- `update` now generates missing, blank, or example `NEXDROP_NODE_KEY` and `NEXDROP_BOOTSTRAP_ADMIN_TOTP_SECRET` values, writes them to `.env`, and displays them immediately.
- Existing Node and OTP keys are preserved permanently and are never regenerated by later updates.

## [2.0.0] - 2026-07-19

### Changed

- Devices now join directly with a Node URL and key; first-party interfaces no longer use pairing codes, challenge IDs, or pending approval.
- Web and desktop experiences now use a chat-style send and history interface, sending to every Node device by default with optional per-device targeting.
- Added image and file drag-and-drop, notification delivery, and one-step `nexdrop://join` onboarding.
- The installer now generates and displays the Node key, administrator credentials, and OTP key.

## [1.0.6] - 2026-07-18

### Changed

- The first valid device on an account became the trust anchor. Later devices entered a pending state and generated a challenge ID, six-digit pairing code, and QR payload automatically.
- Trusted devices could approve new devices on the same account from the device page; pairing was removed from the administration console.

### Fixed

- Fixed the administration console loading several APIs at high frequency and showing an empty page after rate limiting; refresh frequency was reduced and partial failure handling plus manual retry were added.

## [1.0.5] - 2026-07-18

### Changed

- Added automatic trust for devices on the same Node while retaining the existing compatibility pairing flow.
- Web, Windows, Android, Chrome, and Edge now select all trusted devices by default and no longer expose group entry points.
- Added live device and Node presence, per-device traffic statistics, and quick copy and delete actions for messages.

## [1.0.4] - 2026-07-18

### Fixed

- Android Release APKs now use the official apksigner tool and verify both v1 and v2 signatures.
- Standard Flutter CI now verifies both signatures on Release APKs.

## [1.0.3] - 2026-07-17

### Fixed

- Fixed Android package signing compatibility.

## [1.0.2] - 2026-07-17

### Changed

- Improved the production release process and version consistency.

## [1.0.1] - 2026-07-17

### Fixed

- Fixed initial production release issues.

## [1.0.0] - 2026-07-17

### Added

- First stable NexDrop release.
