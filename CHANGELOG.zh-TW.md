# 變更紀錄

[English](CHANGELOG.md)

本檔案記錄 NexDrop 使用者可感知的變更，格式遵循 Keep a Changelog 與語意化版本。

## [Unreleased]

## [2.0.4] - 2026-07-22

### Changed

- 改善雙語文件、API 與即時連線邊界、擴充功能操作效率、Flutter 傳送與連線可靠性、部署診斷，以及 Go 1.26 安全自動化。

## [2.0.3] - 2026-07-22

### Changed

- 將 CodeQL 工作流程統一更新至 4.37.3，並將相關 Dependabot 更新分組，避免元件版本不同步。
- 更新 `docker/setup-buildx-action` 至 4.2.0 與 `actions/checkout` 至 7.0.1。

### Fixed

- 修正自動合併工作流程對 `dependabot[bot]` 與 `github-actions[bot]` 的作者比對，讓可信任的自動更新可在必要檢查通過後合併。
- 解決多個 GitHub Actions 更新同時修改發布工作流程時的衝突。

## [2.0.2] - 2026-07-22

### Changed

- 移除 Web 管理後台、瀏覽器管理 API、配對碼與手動核准流程，節點維運統一由部署命令處理。
- Windows 與 Android 支援 `nexdrop://join` 一鍵帶入節點連結及節點密鑰，設備建立請求會正確傳送節點密鑰。
- Release Workflow 依產品版本推送對應的 `major.minor` 容器標籤，不再固定寫入 `1.0`。
- 可信自動產生的相依更新 PR 會在必要檢查通過後自動合併；功能性 PR 必須同步版本與發布說明。

### Fixed

- 修正 Windows 關閉視窗或選擇完全退出後，系統匣、連線與背景服務仍持續執行的問題。
- 修正設備加入、頁面排版、錯誤提示關閉、生命週期清理及負載測試仍依賴舊核准 API 的問題。

## [2.0.1] - 2026-07-19

### Fixed

- 修正由 1.x 升級至 2.0 時，既有 `.env` 缺少節點密鑰與 Web OTP 密鑰，導致更新後顯示空白的問題。
- `update` 現在會自動產生缺少、空白或仍為範例值的 `NEXDROP_NODE_KEY` 與 `NEXDROP_BOOTSTRAP_ADMIN_TOTP_SECRET`，寫回 `.env` 並立即顯示。
- 已存在的節點密鑰與 OTP 密鑰會永久保留，不會因後續更新而重新產生。

## [2.0.0] - 2026-07-19

### Changed

- 設備改用節點連結與節點密鑰直接加入，第一方介面不再使用配對碼、挑戰 ID 或待核准流程。
- Web 與桌面端整合為聊天室式傳送與歷史介面，預設傳送給節點全部設備，也可只傳給指定設備。
- 支援圖片與檔案拖放、接收通知及 `nexdrop://join` 一鍵導入。
- 安裝流程會產生並輸出節點密鑰、管理員登入資料及 OTP 密鑰。

## [1.0.6] - 2026-07-18

### Changed

- 帳號第一台有效設備作為信任起點；後續新增設備會進入待配對狀態，並由該設備自動產生挑戰 ID、六位數配對碼及 QR 配對資料。
- 已信任設備可在設備頁輸入配對資料核准同帳號的新設備，配對操作不再放在管理後台。

### Fixed

- 修正管理後台高頻同時載入多個 API，觸發速率限制後整頁沒有資料的問題；現在改為較低頻率刷新、局部容錯及手動重試。

## [1.0.5] - 2026-07-18

### Changed

- 同節點裝置自動信任，保留既有配對相容流程。
- Web、Windows、Android、Chrome、Edge 預設全選信任設備並移除群組入口。
- 新增即時裝置／節點狀態、逐設備流量統計、訊息快速複製與刪除。

## [1.0.4] - 2026-07-18

### Fixed

- Android Release APK 使用官方 apksigner 套用並驗證 v1/v2 簽章。
- 一般 Flutter CI 增加 Release APK 雙簽章驗證。

## [1.0.2] - 2026-07-17

### Changed

## [1.0.1] - 2026-07-17
