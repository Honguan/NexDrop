# NexDrop 2.0.0

NexDrop 2.0 將設備加入流程簡化為「節點連結＋節點密鑰」。Windows、Android 與瀏覽器擴充功能不再需要管理員帳號、密碼、OTP、配對碼或人工核准；輸入安裝完成時輸出的節點設定後，設備會直接加入節點並建立自己的加密設備金鑰。

傳送、接收與歷史紀錄整合為聊天室。預設訊息、圖片與檔案會傳送給節點內所有設備，也可以只選擇特定設備；私人訊息只會出現在傳送設備與被選擇設備的聊天室。每筆內容會顯示傳送設備、時間及傳輸狀態，Web 與桌面／Android 支援拖放圖片或檔案。

新設備加入及收到新內容時，Web、Windows、Android 與瀏覽器擴充功能會顯示通知。管理後台可查看在線狀態、最後上線時間，並刪除已離線設備。

Linux 安裝器現在預設使用伺服器 IP 建立 HTTP 節點連結，並輸出節點連結、節點密鑰、一鍵匯入 JSON、Web 管理員帳密、OTP 設定密鑰、目前六位數 OTP 與 Authenticator URI。設定 DNS A／AAAA 記錄後，可將節點連結改成 HTTPS 網域；亦可設定來源 IP 白名單。

## 從 1.x 升級

執行 `./deploy/nexdrop update 2.0.0`。升級會保留資料庫、檔案與既有設備資料，並自動補上節點密鑰及管理員 OTP。舊版客戶端無法使用新的節點密鑰介面，請更新 Windows／Android／Chrome／Edge 後重新貼上 `./deploy/nexdrop credentials` 顯示的一鍵匯入設定。

由於私人對話改為設備層級可見性，升級後每台設備只能查看自己傳送或被指定接收的訊息；傳給所有設備的聊天室內容仍會在所有設備顯示。
