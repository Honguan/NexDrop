# Configuration reference

[繁體中文](configuration.zh-TW.md)

| Variable | Required | Default | Purpose |
|---|---:|---|---|
| `POSTGRES_PASSWORD` | Yes | Randomized by install | At least 16 characters; supports letters, digits, and `. _ ~ ! @ % + , : / -`; the password is not embedded in the database URL |
| `NEXDROP_CURSOR_SECRET` | Yes | None | At least 32 characters; signs history cursors and must remain stable across upgrades |
| `NEXDROP_IMAGE` | No | `ghcr.io/honguan/nexdrop:2.0.4` | Fully qualified Node image tag |
| `NEXDROP_DOMAIN` | Yes | `localhost` | Caddy HTTPS host name |
| `NEXDROP_DATABASE_URL` | Yes in container | Set by Compose | PostgreSQL URL without a password |
| `NEXDROP_DATABASE_PASSWORD` | Yes in container | Set by Compose | PostgreSQL password passed separately so special characters are not parsed as URL syntax |
| `NEXDROP_STORAGE_PATH` | No | `/var/lib/nexdrop` | Encrypted chunk and backup directory |
| `NEXDROP_WEB_PATH` | No | `/usr/share/nexdrop/web` | Web static assets |
| `NEXDROP_MIGRATIONS_PATH` | No | `/usr/share/nexdrop/migrations` | Migration directory |
| `NEXDROP_BOOTSTRAP_ADMIN_*` | First startup | Generated by install | Initial administrator username, email, and password of at least 12 characters; the password supports letters, digits, and `. _ ~ ! @ % + , : / -` |
| `NEXDROP_LOGIN_RATE_LIMIT_PER_MINUTE` | No | `10` | Login attempts per IP and identifier |

`./deploy/nexdrop install` displays every secure default and offers three choices: accept, edit each value, or regenerate every secret. `credentials` normally shows only the administrator's bootstrap login. That initial password becomes invalid after a reset. PostgreSQL and cursor secrets are shown only by `credentials --show-secrets`. Use `configure` for public and bootstrap fields, and `configure-secrets` to replace or regenerate the current administrator password, PostgreSQL password, and cursor secret individually. `.env` changes only after the corresponding database command succeeds. Replacing the cursor secret immediately invalidates all existing cursors. `.env` must remain readable only by the deployment user and must never be committed.
