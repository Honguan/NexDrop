# Changelog

本檔案記錄 NexDrop 使用者可感知的變更，格式遵循 Keep a Changelog 與語意化版本。

## [Unreleased]

## [2.0.0] - 2026-07-19

### Changed

- 設備改用節點連結與節點密鑰直接加入，第一方介面不再使用配對碼、挑戰 ID 或待核准流程。
- Web 與桌面端整合為聊天室式傳送與歷史介面，預設傳送給節點全部設備，也可只傳給指定設備。
- 支援圖片與檔案拖放、接收通知及 `nexdrop://join` 一鍵導入。
- 安裝流程會產生並輸出節點密鑰、管理員登入資料及 OTP 密鑰。

## [1.0.6] - 2026-07-18

### Changed

- 帳號第一台有效設備作為信任起點；後續新增設備會進入待配對狀態，並由該設備自動產生挑戰 ID、六位數配對碼及 QR 配對資料。
- 已信任設備可在設備頁輸入配對資料核准同帳號的新設備，配對操作不再放在管理後台。

### Fixed

- 修正管理後台高頻同時載入多個 API，觸發速率限制後整頁沒有資料的問題；現在改為較低頻率刷新、局部容錯及手動重試。

## [1.0.5] - 2026-07-18

### Changed

- 同節點裝置自動信任，保留既有配對相容流程。
- Web、Windows、Android、Chrome、Edge 預設全選信任設備並移除群組入口。
- 新增即時裝置／節點狀態、逐設備流量統計、訊息快速複製與刪除。

## [1.0.4] - 2026-07-18

### Fixed

- Android Release APK 使用官方 apksigner 套用並驗證 v1/v2 簽章。
- 一般 Flutter CI 增加 Release APK 雙簽章驗證。

## [1.0.3] - 2026-07-17
