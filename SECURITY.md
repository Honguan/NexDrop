# Security policy

[繁體中文](SECURITY.zh-TW.md)

## Supported versions

Security fixes target the latest stable release.

## Report a vulnerability

Use **Report a vulnerability** under GitHub Security Advisories. Do not open a public issue. Include the affected version, reproduction steps, impact, and any practical mitigation. Never attach real user data, tokens, private keys, or file content.

Maintainers acknowledge reports within seven days and provide a remediation and disclosure schedule after triage. Do not publish details before a fixed release is available.

Deployments should pin a complete container version, verify release SHA-256 and attestations, use HTTPS, and regularly test backup restoration.
